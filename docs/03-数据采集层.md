## 一、最终方案：Electron 独立采集应用

### 核心理由

目标网站（小红书等）有强反爬机制，传统 Playwright headless 检测风险高。
Electron 内嵌真实 Chromium，用户手动登录，平台无法区分。

详细方案对比见 [数据采集设计方案.md](./数据采集设计方案.md)。

## 二、三种采集模式

### 模式 1：内嵌浏览器手动采集（主要模式）

```
用户在 Electron 内嵌浏览器中正常浏览
        ↓
看到想要保存的内容
        ↓
点击「保存当前页」按钮
        ↓
内容提取 + 预览弹窗
        ↓
用户确认（可编辑标题/内容）
        ↓
发送到 FastAPI /api/collect/{kb_id}/raw
```

### 模式 2：批量 URL 采集

```
用户粘贴多个 URL（每行一个）
        ↓
Electron 在后台依次访问
（使用已登录的 Session）
        ↓
自动提取内容
        ↓
批量发送到后端
```

### 模式 3：本地文件导入

```
用户拖拽文件到导入区域
（支持 .md / .txt / .pdf / .html）
        ↓
PDF → 提取文本
HTML → readability.js 提取正文
Markdown/TXT → 直接使用
        ↓
上传到 /api/collect/{kb_id}/upload
```

## 三、内容提取策略

### Provider 机制

针对不同网站使用专属解析规则：

```typescript
interface ContentProvider {
  domain: string
  extract(webContents: WebContents): Promise<ExtractResult>
}

interface ExtractResult {
  title: string
  content: string    // Markdown 格式正文
  url: string
  metadata: {
    author?: string
    publishTime?: string
    likes?: number
    collects?: number
    [key: string]: any
  }
}
```

### 内置 Provider 列表

| Provider | 目标网站 | 提取内容 |
|---------|---------|---------|
| `XiaohongshuProvider` | xiaohongshu.com | 帖子正文、互动数据、图片描述 |
| `ZhihuProvider` | zhihu.com | 问题/回答/文章正文 |
| `DefaultProvider` | 所有其他网站 | `@mozilla/readability` 通用提取 |

### DefaultProvider 实现

```typescript
// 在页面中注入 Readability.js 提取正文
const article = await webContents.executeJavaScript(`
  const { Readability } = window.__Readability;
  const reader = new Readability(document.cloneNode(true));
  const article = reader.parse();
  JSON.stringify({
    title: article?.title || document.title,
    content: article?.textContent || '',
    excerpt: article?.excerpt || ''
  });
`);
```

## 四、Cookie 管理

### 策略：Electron Session 自动持久化

- Electron 应用使用独立的 `persist:wiki-collector` Session
- 用户在应用内登录，Cookie 自动持久化到 Electron userData 目录
- 应用重启后保持登录状态
- 不同网站的 Cookie 相互隔离

```typescript
// 创建持久化 Session
const ses = session.fromPartition('persist:wiki-collector')

// 加载页面时使用该 Session
win.webContents.session === ses  // 内嵌浏览器使用同一 Session
```

### Cookie 生命周期

```
首次启动
    ↓
用户在内嵌浏览器中手动访问目标网站
    ↓
正常登录（Cookie 自动保存）
    ↓
后续采集复用该 Cookie
（Cookie 过期时需重新登录）
```

## 五、与 FastAPI 后端通信

```typescript
// 采集端 API 调用
const api = axios.create({
  baseURL: store.settings.apiBaseUrl,  // 默认 http://localhost:8000
  timeout: 30000
})

// 提交采集内容
async function submitDocument(doc: RawDocument) {
  await api.post(`/api/collect/${currentKbId}/raw`, doc)
}

// 批量提交
async function submitBatch(docs: RawDocument[]) {
  await api.post(`/api/collect/${currentKbId}/batch`, { documents: docs })
}
```

## 七、Phase 1 简化方案

> **MVP 阶段先只做本地文件导入，跑通整个后端流水线**

在 Phase 1 中，数据采集通过两种简单方式：

1. **CLI 导入**：
   ```bash
   llm-wiki collect --kb ai-tech --dir ./my-articles/
   llm-wiki collect --kb ai-tech --file article.md
   ```

2. **Web UI 上传**：在流水线管理页面拖拽上传文件

Phase 2 再开发 Electron 采集应用。
