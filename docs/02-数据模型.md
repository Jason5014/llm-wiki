# 数据模型定义

> 系统中所有核心数据结构的完整定义

---

## 一、知识库（Knowledge Base）

### KBConfig — 知识库配置

```python
class KBConfig:
    kb_id: str          # 唯一ID，如 "ai-tech"，只含字母/数字/连字符
    name: str           # 显示名称，如 "AI技术知识库"
    description: str    # 简介
    domain: str         # 领域标签，如 "AI/Tech"、"Finance"
    language: str       # 主要语言，默认 "zh"
    crawl_targets: list[str]   # 爬取目标 URL 列表（可选）
    created_at: datetime
    updated_at: datetime
    status: KBStatus    # creating / ready / processing / error
```

### KBStats — 知识库统计

```python
class KBStats:
    kb_id: str
    raw_count: int      # 原始文档数
    source_count: int   # wiki/source 摘要页数
    entity_count: int   # wiki/entity 实体页数
    concept_count: int  # wiki/concept 概念页数
    indexed: bool       # 是否已建立向量索引
    last_processed: datetime | None
```

## 二、文档流转模型

### RawDocument — 原始文档（Stage 1 输出）

```python
class RawDocument:
    doc_id: str         # 文档唯一ID，由 URL slug + hash 生成
    title: str          # 文档标题
    url: str            # 来源 URL（本地文件为路径）
    content: str        # 完整正文内容（Markdown 格式）
    metadata: dict      # 扩展元数据（作者、时间、点赞数等）
    crawled_at: datetime
    char_count: int
```

### SourcePage — Wiki 摘要页（Stage 2 输出）

```python
class SourcePage:
    doc_id: str
    title: str
    content_type: str   # tutorial / guide / news / opinion / comparison / case-study / other
    url: str
    theme: str          # 2-3句核心主题
    key_points: list[str]       # 4-8条关键要点
    tags: list[str]             # 5-10个关键词标签
    entities: list[str]         # 提及的实体名称列表（如 ["Claude", "OpenAI"]）
    concepts: list[str]         # 提及的概念名称列表（如 ["agent", "mcp"]）
    raw_markdown: str           # 最终输出的 Markdown 内容
    processed_at: datetime
```

### EntityData — 实体数据（Stage 3 中间结果）

```python
class EntityData:
    name: str           # 标准名称，如 "Claude"、"OpenAI"
    category: str       # product / company / tool / person / project / other
    description: str    # 一句话描述
    aliases: list[str]  # 别名列表，如 ["claude", "Claude AI"]
    related_entities: list[str]   # 关联实体名
    related_concepts: list[str]   # 关联概念名
    source_docs: list[str]        # 来源文档 doc_id 列表
```

### ConceptData — 概念数据（Stage 3 中间结果）

```python
class ConceptData:
    name: str           # 标准名称，如 "agent"、"rag"、"工具调用"
    category: str       # technology / method / pattern / theory / term / other
    description: str    # 1-2句描述
    aliases: list[str]  # 别名，如 ["智能体", "AI Agent"]
    related_concepts: list[str]
    related_entities: list[str]
    source_docs: list[str]
```

### WikiPage — 最终 Wiki 页面（Stage 3 输出）

```python
class WikiPage:
    name: str
    page_type: str      # "entity" 或 "concept"
    category: str
    content: str        # 完整的 Markdown 内容（含 YAML frontmatter）
    aliases: list[str]
    related: list[str]  # 所有关联实体/概念
    sources: list[str]  # 来源文档 doc_id 列表
    created_at: datetime
    updated_at: datetime
```

## 三、检索模型

### SearchResult — 单条搜索结果

```python
class SearchResult:
    page_type: str      # "source" | "entity" | "concept"
    name: str           # 页面名称
    score: float        # 相关性分数（0-1）
    snippet: str        # 内容摘录（约200字）
    url: str            # 原始来源 URL（仅 source 类型有）
    path: str           # Wiki 相对路径，如 "entity/Claude"
```

### SearchResponse — 完整搜索响应

```python
class SearchResponse:
    query: str
    answer: str                         # LLM 生成的回答
    sources: list[SearchResult]         # 引用来源（按相关性排序）
    related_entities: list[str]         # 相关实体名
    related_concepts: list[str]         # 相关概念名
    search_time_ms: float
```

## 四、文件存储格式规范

### 4.1 Raw 原始文档格式

`raw/{doc_id}.md`

```markdown
# {标题}

> 抓取时间：2026/05/29 14:30:00

---

## 🔗 来源链接

{URL}

## 📝 正文

{提取的正文内容}

## 📊 元数据（可选）

| 指标 | 数量 |
|------|------|
| ❤️ 点赞 | 139 |
| ⭐ 收藏 | 268 |
```

### 4.2 Wiki Source 摘要格式

`wiki/source/{doc_id}.md`

```markdown
> 原始文档：{doc_id}

---
## 📋 基本信息
- **原标题**：{title}
- **内容类型**：{content_type}

## 🎯 核心主题
{theme - 2-3句话}

## 📌 关键要点
- {point_1}
- {point_2}
- ...

## 🏷️ 关键词标签
#[[标签1]] #[[标签2]] ...

## 🔗 相关实体
[[Entity1]]
[[Entity2]]

## 💡 关联概念
[[concept1]]
[[concept2]]
---
```

### 4.3 Wiki Entity 实体页格式

`wiki/entity/{Name}.md`

```markdown
---
type: entity
category: product | company | tool | person | project
aliases: ["别名1", "别名2"]
tags: []
updated: 2026-05-29
---

# {Name}

> {一行描述}

## 概述
{2-3段综合介绍}

## 核心特性
- **特性1**：说明
- **特性2**：说明

## 典型应用场景
- 场景1
- 场景2

## 关联关系
- **所属公司/项目**：[[Parent]]
- **相关实体**：[[Entity1]]、[[Entity2]]
- **核心概念**：[[Concept1]]、[[Concept2]]

## 参考来源
- [[source/doc_id_1]]
- [[source/doc_id_2]]
```

### 4.4 Wiki Concept 概念页格式

`wiki/concept/{name}.md`

```markdown
---
type: concept
category: technology | method | pattern | theory | term
aliases: ["别名1"]
tags: []
updated: 2026-05-29
---

# {name}

> {一行描述}

## 定义
{精确定义，1-2段}

## 核心原理
{工作原理解释}

## 优势与局限
**优势**
- ...

**局限**
- ...

## 典型应用
- 场景1：[[Entity1]] 中的应用
- 场景2：与 [[concept2]] 结合使用

## 相关概念
- [[RelatedConcept1]] — 关系说明

## 参考来源
- [[source/doc_id_1]]
```

### 4.5 Wiki Index 索引格式

`wiki/index.md`

```markdown
---
type: index
version: 1.0.0
last-updated: 2026-05-29
source-count: 96
entity-count: 25
concept-count: 25
---

# 全局知识库索引

## 一、概念体系目录（concepts）
### 核心技术
- [[concept/agent]] — 智能体，自主执行任务的 AI 系统
...

## 二、实体对象目录（entities）
### AI 模型与产品
- [[entity/Claude]] — Anthropic 旗舰 AI 助手
...

## 三、原始资料索引（sources）
- [[source/doc_id_1]] — 文章标题
...

## 四、别名索引
| 别名 | 指向 |
|------|------|
| 智能体 | [[concept/agent]] |
...

## 五、健康状态
- 原始文档：96 篇
- 概念页面：25 个
- 实体页面：25 个
```

## 五、Milvus Collection Schema

每个知识库对应一个 Milvus Collection：`kb_{kb_id}`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT64，自增主键 | - |
| `vector` | FLOAT_VECTOR(1024) | text-embedding-v3 输出 |
| `doc_id` | VARCHAR(256) | 对应文件名（不含扩展名） |
| `page_type` | VARCHAR(32) | source / entity / concept |
| `title` | VARCHAR(512) | 标题 |
| `content` | VARCHAR(4096) | 文本内容（用于展示 snippet） |
| `chunk_idx` | INT64 | 长文档分块索引，默认 0 |

**索引类型**：IVF_FLAT，度量：COSINE

## 六、SQLite FTS5 Schema

```sql
-- 全文检索表
CREATE VIRTUAL TABLE wiki_fts USING fts5(
    doc_id UNINDEXED,
    page_type UNINDEXED,
    title,
    content,
    tokenize = 'unicode61'
);

-- 元数据表（FTS 搜索后用于获取详情）
CREATE TABLE wiki_meta (
    doc_id     TEXT PRIMARY KEY,
    page_type  TEXT,
    title      TEXT,
    url        TEXT,
    updated    TEXT
);
```
