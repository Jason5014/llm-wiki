## 一、项目背景与目标

### 1.1 要解决的核心问题

传统 RAG 知识库有两个根本缺陷：

1. **知识碎片化**：把文档切成固定大小的 Chunk，丢失上下文、逻辑关系、主题边界
2. **检索浅层化**：Top-K 向量召回，只能做"模糊匹配"，无法进行"导航式推理"

### 1.2 LLM Wiki 的解决思路

参考 Karpathy 提出的 **LLM Wiki** 理念：

- **保留结构**：以"页面"为单位组织知识，而非无意义的 Chunk
- **双向链接**：实体/概念之间建立关联网络，支持多跳推理
- **知识编译**：原始内容经过 LLM 提炼成结构化的 entity / concept 页面
- **导航检索**：LLM 像人类浏览维基百科一样，在知识图谱中主动导航

### 1.3 项目目标

构建一套**企业级/个人级 AI 知识库系统**，具备：

- 🕷️ **多源数据采集**：支持登录网站（小红书等）、公开网页、本地文件
- 🧠 **LLM 驱动的知识处理**：自动提炼摘要、抽取实体/概念、构建关联
- 📚 **Wiki 知识图谱**：结构化的 entity/concept/source 三层知识空间
- 🔍 **混合智能检索**：向量 + 全文 + 图谱 + LLM 推理，四路融合
- 🖥️ **可视化操作界面**：Obsidian 风格的 Wiki 浏览器 + 知识图谱 + 问答界面
- 🔌 **MCP 工具接口**：对外提供标准化的 AI 工具调用能力
- 🗂️ **多知识库管理**：不同领域/用户可以独立建库

## 五、快速启动

### 环境准备

```bash
# 1. 安装所有依赖
make setup

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填写 DASHSCOPE_API_KEY
```

### 一键启动开发环境

```bash
# 启动全部服务（Milvus + FastAPI + Vue + Electron）
./scripts/dev.sh

# 或者跳过 Electron 采集器
./scripts/dev.sh --no-collector

# 停止所有服务
./scripts/stop.sh
```

### 分服务启动

```bash
make milvus     # 启动向量数据库
make api        # 启动 FastAPI 后端（:8000）
make web        # 启动 Vue 前端（:5173）
make collector  # 启动 Electron 采集器
make mcp        # 启动 MCP 服务器（stdio）
```

### 服务地址

| 服务 | 地址 |
|------|------|
| FastAPI 后端 | http://localhost:8000 |
| API 文档（Swagger） | http://localhost:8000/docs |
| Vue 前端 | http://localhost:5173 |
| Milvus 健康检查 | http://localhost:9091/healthz |

### CLI 使用

```bash
# 创建知识库
uv run llm-wiki create my-kb --name "我的知识库"

# 导入文档
uv run llm-wiki collect my-kb --file ./doc.md
uv run llm-wiki collect my-kb --dir ./docs/

# 运行处理流水线
uv run llm-wiki process my-kb

# 搜索
uv run llm-wiki search my-kb "什么是 RAG？"
```

## 六、开发阶段规划

### Phase 1 — 核心流水线 ✅

**目标**：能把本地文件跑完整个流程，在 Web UI 上能搜索和浏览

```
本地文件导入 → LLM 处理 → Wiki 知识图谱 → Milvus 索引 → 搜索问答
```

交付物：
- [x] FastAPI 后端（KB管理 + 流水线 + 搜索）
- [x] LLM 四阶段处理（source/entity/concept/index）
- [x] Milvus + FTS5 检索引擎
- [x] Vue3 基础 UI（KB管理 + Wiki浏览 + 搜索）
- [x] ECharts 知识图谱可视化
- [x] SSE 实时流水线进度
- [x] MCP 服务器（4个工具）
- [x] typer CLI 命令行工具

### Phase 2 — 数据采集 ✅

**目标**：能从目标网站（包括需登录的）采集内容送入知识库

交付物：
- [x] Electron 内嵌浏览器 + 采集控制面板
