# 检索引擎设计

> 混合检索策略：四路融合 + LLM 答案生成

---

## 一、检索策略总览

```
用户输入查询
      │
      ▼
  ┌───────────────────────────────────────────┐
  │               查询分析                     │
  │  解析查询中的实体/概念提及                  │
  └───────────┬───────────────────────────────┘
              │
    ┌─────────┼─────────────┐
    ▼         ▼             ▼
  直接查找   向量检索      全文检索
（精确匹配）（语义相似）  （关键词匹配）
    │         │             │
    └─────────┼─────────────┘
              │
              ▼
          RRF 融合排序
     （Reciprocal Rank Fusion）
              │
              ▼
       取 Top-5 相关页面
              │
              ▼
     读取完整页面内容作为 Context
              │
              ▼
         LLM 生成答案
     （含 [[WikiLink]] 引用）
              │
              ▼
   返回：答案 + 来源列表 + 相关实体/概念
```

## 二、三路检索详解

### 2.1 直接查找（Direct Lookup）

**触发条件**：查询内容与 entity/concept 名称完全匹配或包含关系

**实现逻辑**：
```python
# 从 entity/concept 名称列表中查找
for name in entity_names:
    if query.lower() in name.lower() or name.lower() in query.lower():
        score = 1.0 if exact_match else 0.8
        results.append(SearchResult(page_type="entity", name=name, score=score))
```

**示例**：
- 查询 "什么是 MCP" → 直接命中 `concept/mcp`，score=0.8
- 查询 "Claude" → 直接命中 `entity/Claude`，score=1.0

---

### 2.2 向量检索（Vector Search）

**工具**：Milvus + DashScope text-embedding-v3

**实现逻辑**：
```python
# 1. 对查询向量化
query_vector = await embed(query)  # 1024 维向量

# 2. Milvus 检索
results = milvus_client.search(
    collection_name=f"kb_{kb_id}",
    data=[query_vector],
    limit=10,
    output_fields=["doc_id", "page_type", "title", "content"],
    search_params={"metric_type": "COSINE", "params": {"nprobe": 10}}
)

# 3. 分数：cosine distance → similarity
score = 1.0 - distance  # distance 越小越相似
```

**优势**：捕获语义相似但关键词不同的内容（如查"自主AI系统"能找到"agent"页面）

---

### 2.3 全文检索（FTS Search）

**工具**：SQLite FTS5，`unicode61` 分词器

**实现逻辑**：
```sql
SELECT doc_id, page_type, title,
       snippet(wiki_fts, 3, '...', '...', '...', 20) as snip,
       rank
FROM wiki_fts
WHERE wiki_fts MATCH ?
ORDER BY rank
LIMIT 10
```

**查询预处理**：
- 多词查询用 `OR` 连接（提高召回）
- 转义 FTS5 特殊字符（`"`, `(`, `)` 等）

**优势**：精确关键词匹配，对产品名/命令名效果好（如搜 "/branch" 命令）

## 三、RRF 融合排序

**Reciprocal Rank Fusion** 算法：将多路检索结果合并排序

```python
def rrf_merge(result_lists, top_n=5, k=60):
    """
    RRF score = Σ 1/(k + rank_i)
    k=60 是经验值，减少最高排名对最终分数的过度影响
    """
    rrf_scores = {}
    
    for result_list in result_lists:
        for rank, result in enumerate(result_list, start=1):
            key = f"{result.page_type}:{result.name}"
            rrf_scores[key] = rrf_scores.get(key, 0) + 1.0 / (k + rank)
    
    return sorted_top_n(rrf_scores, top_n)
```

**效果**：出现在多路检索结果中的页面会获得更高的融合分数

## 四、LLM 答案生成

### 4.1 Context 构建

```
取 Top-5 检索结果
  ↓
加载每个页面的完整内容
  ↓
按页面类型标注：
  [ENTITY: Claude]
  [CONCEPT: agent]
  [SOURCE: 15-hermes-features]
  ↓
拼接为 Context（每个页面截取前 800 字）
```

### 4.2 Answer Prompt

```
你是专业的知识库助手。基于以下检索到的知识片段，回答用户问题。

用户问题：{query}

检索内容：
---
[ENTITY: Claude]
Claude 是 Anthropic 开发的大语言模型...

[CONCEPT: agent]
Agent 是能够自主执行任务的 AI 系统...
---

要求：
1. 直接回答，不要说"根据检索结果"等套话
2. 引用概念或实体时用 [[WikiLink]] 标注
3. 知识库中没有足够信息时，诚实说明
4. 简洁有条理，中文回答
```

### 4.3 回答示例

用户问：**"怎么用 Hermes 实现持久化记忆？"**

```
使用 [[Hermes]] 实现持久化记忆主要通过两个文件：

1. **memory.md**：[[agent]] 的记事本，使用 FTS5 + [[llm]] 
   摘要索引，能够召回 8 周内的相关历史记忆。

2. **USER.md**：存储用户个人档案，跨所有会话和平台生效。

配置完成后，[[Hermes]] 会在每次会话启动时自动读取这两个
文件，无需每次重新介绍自己。

> 参考来源：[[source/15-hermes-features]]、[[source/hermes-memory-guide]]
```

## 五、图谱导航检索（高级，Phase 3）

**思路**：Agentic Retrieval 模式，LLM 自主决定下一步检索路径

```
用户问题
    ↓
LLM 分析：需要哪些知识点？
    ↓
检索入口页（entity/concept）
    ↓
读取关联链接（[[WikiLink]]）
    ↓
决定是否需要继续跳转
    ↓
汇总信息生成答案
```

**实现方式**：ReAct Pattern（Thought → Action → Observation 循环）
